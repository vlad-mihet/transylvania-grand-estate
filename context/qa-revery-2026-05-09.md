## How to re-run

Stack must be up first (`docker compose up -d`, then `pnpm --filter @tge/api dev` and `pnpm --filter @tge/revery dev`). Then:

```bash
bash scripts/qa-smoke.sh                                         # API + page smoke (full)
pnpm --filter @tge/revery test:e2e                               # full Playwright suite (3 browsers)
pnpm --filter @tge/revery test:e2e -- --project=chromium         # Chromium only
pnpm --filter @tge/revery test:e2e:headed                        # for exploratory observation
pnpm --filter @tge/revery test:e2e:report                        # open last HTML report
```

Specs are parameterised at file load time, so one suite covers all four locales. Each spec lives at `apps/revery/tests/e2e/<area>.spec.ts` and shares `_fixtures/{locale,routes,api,a11y}.ts`.

## New findings — open

### 🔶 Critical #16 — `fr.json` and `de.json` are paste-of-`en.json` for 9+ public sections

**App:** revery.
**Where:** `apps/revery/messages/fr.json`, `apps/revery/messages/de.json`. Sections affected: `ContactPage`, `CitiesPage`, `CityDetailPage`, `DevelopersPage`, `DeveloperDetailPage`, `FAQPage`, `ArticleDetailPage`, `AboutPage`, `InquiryModal`, `Common`. (Detection: `node`-side `deepEq()` of every top-level section between locales.)
**Repro:** visit `/fr/contact` or `/de/faq`. The `<h1>` and form labels render English. The locale-aware nav and footer translate (those keys are populated), exposing the drift even more sharply.
**Expected:** every `<h1>` / form-label / call-to-action shows native French / German.
**Actual:** every leaf string in those sections is identical to `en.json`. Translators or a translation-vendor pass missed these areas — `Wave 3.6` (memory `project_qa_phase2.md`) covered admin only.
**Fix area:** dedicated translator pass. **Out of scope for this engagement** (machine translations would risk worse quality than the visible drift). Locked by `i18n.spec.ts:46` (fr) and `:57` (de) so the suite goes red until each section is translated.
**Severity rationale:** every customer landing on `/fr/*` or `/de/*` from the language switcher sees an English contact / about / FAQ page. Brand-credibility blocker for those locales but not a complete failure (page renders, forms submit).

### 🔸 Major #16 — `@Throttle({ default: { ttl: 60_000, limit: 5 } })` on `POST /inquiries` does not enforce

**App:** api.
**Where:** `apps/api/src/inquiries/inquiries.controller.ts:53`. Decorator imported from `@nestjs/throttler@6.5.0`. Global throttler module at `apps/api/src/app.module.ts:131-137`.
**Repro:**
```bash
for i in {1..15}; do
  (curl -s -X POST http://localhost:4000/api/v1/inquiries \
    -H "Content-Type: application/json" -H "X-Site: REVERY" \
    -d "{\"name\":\"P$i\",\"email\":\"p$i@x.y\",\"phone\":\"+40\",\"message\":\"Probe number $i with enough chars\",\"type\":\"general\"}" \
    -o /dev/null -w "%{http_code} ") &
done; wait; echo
# → 201 201 201 201 201 201 201 201 201 201 201 201 201 201 201
```
Sequential and parallel bursts both succeed; 15 inquiries in ~5 s, zero `429`s. Same pattern with `POST /auth/login` (7 attempts → all `401`, no `429`). `DEV_AUTH_THROTTLE_DISABLED` is not set in `apps/api/.env` (commented out) and `skipIf` shouldn't fire. `qa-smoke.sh` Phase B.5 reported `Inquiries hit 429 within 6 attempts` earlier in this same session, then later runs no longer trigger it — points at storage state corruption rather than config.
**Expected:** 6th request within a 60-second window returns `429 ThrottlerException: Too Many Requests`.
**Actual:** all requests pass through. The throttler appears to be silently degraded.
**Fix area:** `@nestjs/throttler` storage / module config in `apps/api/src/app.module.ts:131`. Likely a `@nestjs/throttler@6.5.0`-specific quirk with the in-memory storage TTL prune — needs runtime instrumentation: log `ThrottlerStorage.increment` calls and verify the `key`/`ttl` keys match across requests. Worth checking `getTracker` / `trust proxy` interaction too — Express's `req.ip` should resolve to `X-Forwarded-For[0]` per `main.ts:41`, but the test bursts work both *with* and *without* an XFF header, suggesting the tracker is consistent and the storage layer is the actual fault.
**Severity rationale:** Major (not Critical) because the API doesn't crash or leak; the only impact is that automated submission bots can spam the inquiry endpoint and pollute the admin queue. Locked by `rate-limit.spec.ts` flagged with `test.fail()` so the suite goes green once the throttler returns to enforcing.

## New findings — fixed inline (this session)

Each of the following had a regression spec authored before the fix and re-verified green in the final run.

### Major #17 — Mobile nav trigger missing accessible name

**Was:** `apps/revery/src/components/layout/mobile-nav.tsx:37` rendered `<Button variant="ghost" size="icon"><Menu/></Button>` with no `aria-label`. axe-core: `[critical] button-name`.
**Fix:** added `aria-label={t("openMenu")}`; the `<Menu>` icon got `aria-hidden="true"`. `Navigation.openMenu` was previously `null` in all 4 locales — backfilled with `Deschide meniul / Open menu / Ouvrir le menu / Menü öffnen`.
**Locked-by:** `responsive.spec.ts` "375px viewport reveals a mobile-nav trigger".

### Major #18 — Carousel scroll buttons missing accessible names (×4)

**Was:** `featured-properties.tsx:54-67` and `recent-articles.tsx:51-64` each render two `<button>` chevron-only triggers without `aria-label`. axe-core: `[critical] button-name`. Four buttons total flagged on every home-page render.
**Fix:** added `aria-label={t("scrollPrevious"|"scrollNext")}` and `aria-hidden="true"` on the inner `<ChevronLeft|Right>` icons. New keys `HomePage.featured.scroll{Previous,Next}` + `HomePage.articles.scroll{Previous,Next}` backfilled across all 4 locales.
**Locked-by:** `a11y.spec.ts` home test.

### Major #19 — Contact form `<Label>` not associated with inputs

**Was:** `apps/revery/src/components/contact/contact-form.tsx` rendered `<Label>` as a sibling of `<Input>` with neither `htmlFor` nor a wrapper relationship — five fields total (name, email, phone, budget, message) flagged by axe `[critical] label`.
**Fix:** added `id="contact-{field}"` on every input/textarea/select and matching `htmlFor` on every label. The `<SelectTrigger>` for the budget dropdown also got `aria-label={t("form.budget")}` (Radix Select doesn't infer label from sibling).
**Locked-by:** `a11y.spec.ts` contact test.

### Major #20 — Filter-bar `<SelectTrigger>` instances missing accessible names (×4)

**Was:** `property-filter-bar.tsx:251, 271, 305` (Type / Transaction / Radius) and `property-list-view.tsx:63` (Sort). All four exposed `<SelectValue placeholder>` only — axe-core treats Radix combobox buttons as needing an explicit `aria-label`.
**Fix:** added `aria-label` to each, sourced from the same translation key already rendered as the visible `<p>` label above the select (so the visible label and the screen-reader label match).
**Locked-by:** `a11y.spec.ts` properties-list test.

### Serious #21 — About page step numbers fail color-contrast

**Was:** `apps/revery/src/app/[locale]/about/page.tsx:208` rendered the decorative numbers `01 / 02 / 03` with `text-border` (resolves to `#E5E7EB` — ~1.3:1 against `bg-background` white). axe-core: `[serious] color-contrast`.
**Fix:** swapped to `text-muted-foreground` (`#4B5563`, ~7.5:1) and added `aria-hidden="true"` since the numbers are purely ornamental — the meaningful content is the adjacent `<h3>` and description.
**Locked-by:** `a11y.spec.ts` about test.

### Serious #22 — `InquiryTrigger` produced nested-interactive ARIA violation

**Was:** `packages/ui/src/components/inquiry/inquiry-trigger.tsx` wrapped its children in `<div role="button" tabindex="0">`. All five usages in `apps/landing` and three in `apps/revery` pass a `<Button>` / `<AccentButton>` as the child — producing a `<div role="button">` containing a real `<button>` (axe `[serious] nested-interactive`). Two interactive controls nested means screen-reader users hit two announcements per trigger.
**Fix:** refactored the component to use `Slot.Root` from `radix-ui` so the click handler is merged onto the child element directly; no wrapper div is rendered. All call sites continue to work without changes (every existing usage already passed a single interactive child). Verified manually that `apps/landing` and `apps/revery` both compile and the inquiry modal still opens.
**Locked-by:** `a11y.spec.ts` home + properties-list + property-detail tests.

### Trivial — qa-smoke.sh + qa-runbook.md stale port defaults

**Was:** `scripts/qa-smoke.sh` defaulted `QA_API_URL=http://localhost:3333/api/v1`, `QA_LANDING_URL=http://localhost:3000`, `QA_ADMIN_URL=http://localhost:3001`, `QA_REVERY_URL=http://localhost:3002`, plus four hard-coded `:3333` references in CORS/Swagger/uploads probes. Actual ports flipped to **`:4000` / `:3050` / `:3051` / `:3052`** during Phase-2 Wave 3 Pass A (memory `project_qa_phase2.md`). The runbook (`docs/qa-runbook.md`) had matching drift in three places.
**Fix:** corrected all defaults and the CORS preflight assertion (origin `:3000` → `:3050`). Smoke flipped from "API unreachable" + 12 page-smoke skips to a clean baseline.
**Locked-by:** smoke baseline now passes 90/0/2 by default — no env overrides needed.
